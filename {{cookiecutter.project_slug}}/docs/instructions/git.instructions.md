# Commits

1. Follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) specification for commit messages.
1. Group related changes into a single commit. Avoid making multiple commits for small, related changes.
1. Perform commits in a logical order, following a clear progression of changes. For example, if you are implementing a new feature, start with commits that set up the necessary infrastructure before committing the actual feature implementation.
1. Do not commit unless specifically requested to do so.
